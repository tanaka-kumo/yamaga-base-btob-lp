## 0. これは何のプロジェクトか（30秒サマリー）

熊本県山鹿市の廃校活用施設 **YAMAGA BASE**（運営：やまがBASE株式会社）の
**BtoB（法人）向け集客LP** を制作・運用するプロジェクト。
制作主体は **一朶の雲株式会社**（YAMAGA BASE側のパートナー）。

- **成果物形態**：独立した静的LP（単体HTML + CSS + 最小限のJS / 別ホスティング想定）
- **中心読者**：熊本県内の経営者・経営企画担当者
- **LPの主役**：①経営合宿での利用　②中原代表の経営セミナー（MBAエッセンス）
- 既存コーポレートサイト（https://www.yamagabase.com/ ）とは別物として作るが、
  世界観・ブランド（bY）の連続性は保つ。

---

## 1. 最重要・解決すべき課題（これがLPの存在理由）

既存サイトの分析から導いた、このLPが解かねばならない課題：

1. **認知の「質」が足りない** — 名前は知られていても「自社の何の課題を、ここでどう解決できるか」を
   企業担当者がイメージできない。→ **施設機能の羅列ではなく「課題 → 利用シーン → 効果」への翻訳**を行う。
2. **稟議に乗らない** — 情緒的な世界観は強いが、意思決定者が社内提案するための
   「効果・根拠・利用イメージ・費用感」が不足。→ **BtoBの意思決定に効く要素**を必ず置く。
3. **ワクワクが伝わりきらない** — 写真は多いが体験価値の解像度が低い。
   → 「廃校×秘密基地」という唯一無二の非日常を、合宿/研修の文脈で体感的に訴求する。

> 制作の合言葉：**「ワクワク」と「稟議が通る理性」の両立。** どちらか一方では失敗。

---

## 2. ぜったいに守るルール（Do / Don't）

### Do
- 主役2軸（経営合宿・経営セミナー）を**ファーストビューと構成の中心**に置く。
- すべてのセクションを「読者（県内経営者）の課題」起点で書く。機能起点で書かない。
- CTAは**問い合わせ／資料請求／視察予約**へ自然に収束させる。複数CTAを散らさない。
- 既存ブランド資産（bY、be you / beyond you / beside you、iReaction、「秘密基地」）を尊重する。
- 数値・実績・固有名詞は `docs/facts-and-figures.md` に集約し、**未確認の数字を勝手に創作しない**。

### Don't
- 既存サイトのテキストを**コピペ／20語以上の引用をしない**（参考にしつつ自分の言葉で書く）。
- 確認の取れていない料金・実績・社名（導入企業名等）を**断定で書かない**。プレースホルダにする。
- `reference/` 内の取得済みコンテンツを成果物にそのまま貼らない（あくまで一次情報の控え）。
- 個人情報・問い合わせ先メールアドレス等を勝手に確定しない。`docs/facts-and-figures.md` の確定値のみ使う。

## 4. 作業の進め方（推奨フロー）

1. `docs/01`〜`docs/05` を読み、`docs/facts-and-figures.md` で「確定 / 要確認」を把握する。
2. コピーを書くときは `prompts/copywriting-prompt.md` を使い、`docs/02` の翻訳表に従う。
3. 実装は `prompts/build-lp-prompt.md` と `docs/03`（構成）・`docs/04`（デザイン）に従う。
4. 仕様・方針を変えたら **必ず `docs/decision-log.md` に追記**してから進める。
5. 数字や固有名詞で迷ったら創作せず `docs/facts-and-figures.md` の【要確認】に積み、本文はプレースホルダにする。

---

## 5. 技術メモ（静的LP前提）

- 単体で完結する `index.html` を基本とし、外部依存は最小に（Webフォント・軽量JSのみ）。
- 日本語Webフォントは Noto Sans JP 等を想定（Google Fonts もしくはセルフホスト）。
- レスポンシブ必須（県内経営者はモバイル閲覧も多い）。モバイルファーストで設計。
- 計測タグ（GA4等）を後から差せるよう `<!-- analytics -->` のフックを残す。
- アクセシビリティ（コントラスト・代替テキスト・見出し階層）を最低限担保する。
