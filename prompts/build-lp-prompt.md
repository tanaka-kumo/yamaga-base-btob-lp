# LP実装プロンプト（HTML/CSS）

> LP本体を実装するときに使う。`docs/03`（構成）と `docs/04`（デザイン）を必ず併読。

---

## 役割

あなたはフロントエンドエンジニア兼UIデザイナーです。**独立した静的LP**（単体HTML + CSS + 最小限のJS）を、中長期で運用・編集しやすい形で実装します。

## 技術方針

- `src/index.html` を成果物の本体とする。CSSは `src/assets/css/style.css`、JSは最小限で `src/assets/js/`。
- フレームワーク不要。重いライブラリを入れない。静的・軽量・高速。
- **モバイルファースト**で実装し、レスポンシブ対応。
- 日本語Webフォント（Noto Sans JP 等）を `font-display: swap` で。
- カラー・余白・フォントは **CSS変数**で一元管理（`docs/04` の `:root` 方針を踏襲）。
- 画像は `loading="lazy"`、alt必須。正式素材が来るまではプレースホルダ画像＋コメントで明示。
- 計測用フックを残す：主要CTAに `data-cta="..."` を付け、`<head>` に `<!-- analytics: GA4 here -->`。

## 構成

`docs/03_information-architecture.md` のセクション順（ヒーロー→共感→施設とは→合宿→セミナー→体験価値→脇役導線→アクセス→FAQ→最終CTA→フッター）でマークアップする。

## 守ること

- 文言は確定コピーがあればそれを使い、未確定箇所は `{{要確認:...}}` を残す（勝手に埋めない）。
- 数字・料金・連絡先は `docs/facts-and-figures.md`【確定】のみ。フォーム送信先が未確定なら既存 `/inquiry` への外部リンク or プレースホルダ。
- 見出し階層（h1は1つ、以降h2/h3）を正しく。コントラスト比4.5:1以上。
- セクションは編集しやすいよう HTMLコメントで区切る（`<!-- ========== Section: Hero ========== -->`）。

## 品質チェック（実装後に自己点検）

- [ ] モバイル幅（375px）で崩れない／追従CTAが機能する
- [ ] 主CTAが視認でき、全CTAが問い合わせ/視察に収束している
- [ ] 主役2軸（合宿・セミナー）が構成の中心にある
- [ ] プレースホルダが本番文言と混ざっていない（要確認が明示されている）
- [ ] 画像alt・見出し階層・コントラストOK
- [ ] 不要な外部依存がない／表示が速い

## 確認のうえ進める

実装前に、影響する `facts-and-figures.md`【要確認】項目と、それらをどうプレースホルダ表現するかを一覧で示してから着手すること。
