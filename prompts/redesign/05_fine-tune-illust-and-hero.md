# 05 微調整：イラスト切り抜き・吹き出し詰め・ヒーロー題字の白背景

> `==== 貼り付け ====` の中だけを Claude Code に貼る。

==== 貼り付け ====

モック準拠で実装したLPに、見た目の微調整を3点加えます。配色・トンマナ・他セクションは変えないこと。

## 1. イラスト画像の透明余白を切り抜いて、枠いっぱいに収める

LP用イラスト（`public/assets/` 配下、元 `LP用画像` の 1〜10.png）は、**絵の周囲に大きな透明余白**が入っている（例：1000×1000 の中に絵は 373×633 ほど）。これが原因で、枠の中で絵が小さく見えている。

- **対応**：各PNGの**アルファチャンネルの不透明範囲（bounding box）で自動トリミング**し、余白を削る。中の絵は変えない。
  - Python/Pillow の例：
    ```python
    from PIL import Image
    im = Image.open(src).convert("RGBA")
    bbox = im.split()[3].getbbox()      # 不透明領域の範囲
    im.crop(bbox).save(dst)             # 余白を切り落として保存
    ```
  - ImageMagick なら `convert in.png -trim +repage out.png`。
  - 元ファイルは上書きせず `*-trim.png` 等で書き出し、参照を差し替える方針でよい（対応表をコメントで残す）。対象は共感の 1〜4 と、事業カードの 5〜10。
- **CSS** は、切り抜き後の画像が**余白ほぼゼロで大きく収まる**よう調整：
  - 共感の人物アイコン：`object-fit:contain` のまま、アイコン枠を一回り大きく。画像が縦長になるので**枠も縦長**にして下揃え。
  - 事業カード／スポット画像（5〜10）：枠内側の `padding` をほぼ 0 にして画像を最大化。`overflow:hidden` ではみ出し防止。色付き背景パネルはそのまま。

## 2. 共感セクションの吹き出しの余白を詰める

「その学び、半年後も〜」の各吹き出しに、**1行分の空き余白**ができている（イラストと高さを揃えるため吹き出しを縦に引き伸ばしているのが原因）。

- 吹き出しは**中のテキストにフィット**させ、縦に伸ばさない。`.chat` の `align-items` を `stretch` → `flex-end`（または `center`）に変更。イラストはテキストより多少背が高くてOK。
- 吹き出し内の上下 `padding` を詰める。余分な行間があれば詰める。

## 3. ヒーロー題字に「白い背景ボックス」を付ける

「世界水準の経営学を、自社のチームに。」を、**各行に白い背景ボックス**が付くスタイルにする（参考画像と同じ。文字＝濃色、背景＝白、行ごとに文字幅でフィット）。

- 実装：テキストを `<span>` で囲み、次を適用——
  ```css
  background:#fff; color:var(--ink);
  box-decoration-break:clone; -webkit-box-decoration-break:clone;
  padding:.08em .3em; line-height:1.7; border-radius:4px;
  ```
  改行ごとに白ブロックが付く（`box-decoration-break:clone` が肝）。背景動画の上でも読みやすくなる。
- 行間が詰まりすぎないよう `line-height` で各行の白ブロックに隙間を作る。

## 守ること
- トンマナ・配色・レイアウト・他セクションは変更しない。本文コピー・計測属性（`data-cta` 等）・`{{要確認}}` は維持。
- モバイル 375〜430px で崩れないこと。各変更後にスクリーンショットで確認する。

まず「3点の変更方針」と「トリミング対象ファイル一覧」を提示してから着手すること。
