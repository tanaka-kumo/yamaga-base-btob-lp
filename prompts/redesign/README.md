# リデザイン用バイブコーディング・プロンプト集

参考サイト（ボーダレスアカデミー）の**質感・温度**を、YAMAGA BASE のブランドに翻訳して
今のLPに寄せていくための、Claude Code 用プロンプト一式。

## ファイル構成

| ファイル | 役割 | いつ使う |
|---|---|---|
| `03_borderless-clone-spec.md` | **★いま使うべき本命**。参考サイトを数値で固定した模倣スペック（フォント・配色・ボタン・カードを hex/px/weight で指定） | 「参考に寄らない／ほぼ変わらない」とき。これ単体で土台〜部品まで作り直せる |
| `00_design-direction.md` | 北極星。トークン・コンポーネント語彙・Do/Don't | 毎回いちばん最初に読ませる。方向性を変えたいときはここだけ編集 |
| `01_phase1-master-prompt.md` | 土台づくり（抽象寄り・質感方針） | `03` が刺さらないとき or 穏当に寄せたいとき |
| `02_section-prompt-library.md` | セクション別のコピペ用プロンプト（A〜J） | 土台のあと、1ブロックずつ流す |

> **迷ったら `03` を使う。** 前回「ほぼ変わらない」結果になった原因は指示が抽象的だったため。
> `03` は色・角丸・フォント weight まで数値で固定し、「微調整ではなく作り直し」と明示しているので、参考サイトに寄りやすい。

## 進め方（おすすめ）

1. Claude Code を開き、まず **`00` を読ませる**。
2. **`01`（Phase1マスター）** を貼って土台を作る。ステップごとに止まるので差分を確認しながら進む。
3. 土台が固まったら、**`02` の A→B→C…** を1つずつ流す。気になる所だけでもOK。
4. 大きな構成変更（Phase2）に進むときは `docs/decision-log.md` に記録してから。

## 編集して育てるコツ

- **色や角丸を変えたい** → `00_design-direction.md §3.1` の値だけ書き換える。全プロンプトに自動で効く。
- **改修したい箇所が増えた** → `02` に同じ書式でブロックを追記する。
- **写真** → いまはフリー素材（picsum）で仮当て。本番素材が来たら `data-placeholder="true"` の箇所を順に差し替え。

## 前提（このリポジトリの技術スタック）

Next.js（App Router）+ Tailwind v4 + shadcn/ui。トークンは `src/app/globals.css` の `:root` と `@theme inline` で管理。
セクションは `src/components/sections/*`、共通部品は `src/components/site/*` と `src/components/ui/*`。

## 守るルール（既存ルールの要約）

- Phase1 は**質感だけ**。構成・順番・確定コピーは変えない。
- 未確認の数字・料金・社名・実績は**創作しない**（`{{要確認:...}}` のまま器だけ整える）。
- ブランド資産（bY、be you / beyond you / beside you、六角形、「秘密基地」、灯籠）を尊重。
- CTA は問い合わせ／資料請求／視察予約に収束。
