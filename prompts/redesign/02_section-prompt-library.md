# 02 セクション別プロンプト集（コピペ用ライブラリ）

> **使い方**：`01_phase1-master-prompt.md` の土台づくりが終わったあと、ここから**1ブロックずつ**コピーして流す。
> どのブロックも、冒頭で `prompts/redesign/00_design-direction.md` を参照させる前提。
> 各ブロックは独立しているので、好きな順・好きなものだけ実行できる（後からの差し替え・再実行が容易）。
>
> 共通の前置き（毎回これを先頭に付けると安定）：
> 「`prompts/redesign/00_design-direction.md` の方向性に従い、Phase1（質感のみ・構成とコピーは維持）として、以下を改修してください。実装前に方針を一言で確認し、差分を見せてから進めてください。」

---

## A. ヒーロー（`src/components/sections/hero.tsx`）

```
ヒーローを「写真主役＋温かいグラデ」に作り変えてください（構成・見出しコピーは維持）。
- 背景：濃色グラデのままでなく、施設写真（フリー素材 picsum 仮当て＋差し替えコメント）を主役にし、
  上に黒に近い濃色グラデの薄膜を重ねて文字可読性を確保。MediaFrame ではなく背景レイヤとして配置可。
- レイアウト：見出し・リード・CTA を左寄せ、写真は背景 or 右側。モバイルは縦積み。
- Eyebrow「YAMAGA BASE｜法人向け 経営研修・事業開発」はゴールドのミニラベル＋Pill 風に。
- 現状の破線「象徴ビジュアル」空箱は削除し、本物の写真枠に置換。
- CTA：主ボタン（相談する）はゴールド寄りの存在感、副ボタンはアウトラインのまま。タップ48px以上。
- 直下に、次セクションへ繋がる実績メダル行（StatMedallion）を置けるスペースを用意（数値は {{要確認}} のプレースホルダで可）。
コントラスト4.5:1以上、alt 必須、ヒーロー画像のみ loading=eager 可。
```

---

## B. 実績メダル（`src/components/sections/results.tsx`）

```
実績を「エンブレム（メダル）型」で見せられる枠に作り変えてください。
- StatMedallion 部品を新規作成：円 or 月桂樹風の囲み＋大きな数字＋小ラベル、色はゴールド系。
- 数値は未確認のため {{要確認:受講満足度}} {{要確認:導入社数}} などのプレースホルダを入れ、
  flags.showStats が false の間は「掲載準備中」の整った注記を出す（今のロジックは壊さない）。
- 「世界水準の知見を、確かな体制で。」の見出し・リードは維持。
- ロゴ枠は破線をやめ、淡いトーンの整ったプレースホルダ枠＋許諾待ち注記に。
数字を勝手に創作しないこと。あくまで見せ方（器）の用意。
```

---

## C. 提供内容カード（`src/components/sections/overview.tsx`）

```
3つのサービスカードを SoftCard 基準に質感アップしてください（文言・リンクは維持）。
- 角丸2xl・shadow-soft・ホバーで shadow-lift＋微小 translateY。
- 「① 主役」等のタグは Pill バッジ化（主役カードはゴールドの強めPill＋上辺アクセント）。
- 各カードに小さなアイコンチップ（lucide）を1つ添えて視認性を上げる（装飾過多にしない）。
- 下部の補足ボックスは、淡いセージ（tint-green）の角丸ボックスに。
3カラム→モバイル1カラムの並びは維持。
```

---

## D. POINT / FEATURE カード（特徴・強みの列挙セクション全般）

> 対象例：`training.tsx` `seminar.tsx` `ventures.tsx` などで「特徴を複数並べる」箇所。

```
特徴を並べる箇所を、参考サイトの「POINT / FEATURE」カードに作り変えてください。
- FeatureCard 部品を新規作成：左上に小さな丸「POINT」バッジ（ゴールド地・白文字）、
  右上に斜体の極太数字「FEATURE 01」、その下に太字タイトル＋本文。
- カードは白地・角丸2xl・shadow-soft。番号は 01,02,03… のゼロ埋め。
- 既存の見出し・本文コピーは維持し、器だけ差し替える。
どのセクションに適用したかを報告すること。文言は創作しない。
```

---

## E. こんな方へおすすめ（FitBox）

> 対象：合宿/セミナー/サービスで「こういう企業・人に向く」と列挙している箇所があれば。

```
「こんな方へおすすめ」ボックス（FitBox）を作成・適用してください。
- 淡い枠（ゴールド or セージの細枠）＋上部に見出しバー「こんな企業様におすすめ」。
- 中身は緑チェック（lucide CheckCircle）の箇条書き。各項目1〜2行。
- 該当コピーが docs にある場合のみ使用。なければ {{要確認:対象像}} のプレースホルダ3点で器だけ用意。
```

---

## F. プロセス（`src/components/sections/process.tsx`）

```
「はじめ方」のステップを、温かいタイムライン表現に整えてください（5ステップ・文言は維持）。
- 各ステップ：丸番号バッジ（ゴールド）＋ラベル。PCは横並び、ステップ間を細い点線コネクタで繋ぐ。
- 現状の単調な枠を、SoftCard 寄りの質感に。番号は斜体太字で装飾的に。
- 「費用：…お見積り」の注記は維持。
```

---

## G. 講師紹介（`src/components/sections/instructor.tsx`）

```
講師紹介を、信頼感の出るプロフィールカードにしてください（肩書・経歴コピーは維持）。
- 左に円形の顔写真枠（MediaFrame・フリー素材仮当て＋差し替えコメント）、右に氏名・肩書・略歴。
- 肩書は Pill バッジ（例：HBS MBA）。背景は paper か淡 tint で他セクションと差をつける。
- 数値・実績の創作はしない。未確認は {{要確認}}。
```

---

## H. 最終CTA（`src/components/sections/final-cta.tsx`）

```
最終CTAを、参考サイト風の「濃色CTAパネル」に作り変えてください（見出し・リードは維持）。
- 背景：ディープグリーン（--brand-green-d）の角丸2xl大パネル（アンバーのベタ塗りをやめる、または併用は控えめに）。
- 中央に白の大見出し、その下にリード。主ボタンは「ゴールドの丸ピル＋右矢印（lucide ArrowRight）」。
- 「無料相談」等の Pill タグを見出し上に添える（事実に反しない範囲で。料金条件が未確認なら付けない）。
- 「お問い合わせ窓口は準備中…」の注記は維持。CTAリンク先・data-cta は変えない。
```

---

## I. ヘッダー / フッター / 追従CTA（`site/header.tsx` `site/footer.tsx` `site/mobile-cta.tsx`）

```
ヘッダー・フッター・追従CTAの質感を全体トーンに合わせてください。
- ヘッダー：スクロールで半透明クリーム＋微ブラー、bY マークを尊重。リンクは控えめ、CTAだけ強調。
- フッター：ディープグリーン or 墨地に、bY とタグライン（be You. beyond You.…）を品よく。
- 追従CTA（モバイル）：ゴールド丸ピル＋影で指に収まるサイズ。既存の計測属性は維持。
ブランド資産（bY・六角形）を壊さないこと。
```

---

## J. FAQ（`src/components/sections/faq.tsx`）

```
FAQ（アコーディオン）の質感を整えてください（設問・回答は維持）。
- 各項目を白の SoftCard 風に、開閉アイコンをゴールドに。区切りをやわらかく。
- 余白を広げ、文字を読みやすく（本文 16px・行間 1.8）。
shadcn の accordion 構造は活かしてよい。
```

### 使い終わったら

- 大きめの方針変更（例：構成そのものを変える＝Phase2 へ）を決めたら、`docs/decision-log.md` に必ず追記。
- 新しく増やしたいセクション改修があれば、このファイルに**同じ書式でブロックを追記**すれば資産が育つ。
