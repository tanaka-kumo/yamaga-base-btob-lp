# 00 リデザイン方向性（北極星 / Design Direction）

> このファイルは「参考サイト（ボーダレスアカデミー）の質感を、YAMAGA BASE のブランドに翻訳した設計図」です。
> Phase1／セクション別プロンプトを流すとき、**毎回このファイルを冒頭に読ませてから**作業させてください。
> 値（色・角丸・影など）はすべてここに集約してあるので、**方向性を変えたいときはまずこのファイルだけを編集**すれば全体に効きます。

---

## 0. このリデザインのゴール（1行）

今の「Claude Code で作った汎用テンプレ感」を脱し、参考サイトのような
**「やわらかく・温かく・作り込まれた信頼感」** を、YAMAGA BASE の世界観（廃校×秘密基地）で表現する。

**Phase1 のスコープ＝質感・温度だけ寄せる。** セクションの構成・順番・コピーは原則いじらない。
（レイアウト構造の変更は、質感が固まったあとに別フェーズで検討する。写真は今はフリー素材で仮当て。）

---

## 1. 「Claude Code 感」の正体と、つぶす対象

今の LP が素人っぽく見える具体的な原因。Phase1 はここを潰す。

| 症状 | 今 | 直す方向 |
|---|---|---|
| 画像が破線の空箱 | `border-dashed` のプレースホルダ | **写真枠**（フリー素材＋角丸＋やわらかい影）に置換 |
| カードが平板 | 細い線＋影なし | **やわらかい影＋大きめ角丸＋ホバーで微浮上** |
| 見出しが単調 | Eyebrow＋太字 h2 の繰り返し | **英語ミニラベル（ゴールド）＋ピル型バッジ**でリズム |
| 余白が窮屈 | セクション padding が小さめ | **縦の余白を広く**、息をさせる |
| 配色が硬い | 濃いネイビー基調＋アンバー | **クリーム下地＋セージ／ゴールド**で体温を上げる |
| 装飾がない | bY・六角形・灯籠モチーフが未使用 | 控えめな**装飾モチーフ**を差す（やりすぎ禁止） |
| 証拠がない | 実績が非表示 or テキスト | **エンブレム（メダル）型の数値**で見せる枠を用意 |

## 2. 参考サイトのデザインDNA（分解）

参考サイトから「盗む」具体的な作法。**コピペではなく作法を真似る**。

1. **温かいクリーム下地**＋セクションごとに「白／淡いトーン」を交互に敷いてリズムを作る。
2. **ヒーローは写真主役**。やわらかいグラデの上に、黒に近い濃色の大見出し。
3. **実績はエンブレム（メダル）**。月桂樹風の囲み＋大きな数字（94% / 130名 / 1,600名…の見せ方）。
4. **角丸が大きい**（16〜24px）。影はやわらかく低コントラスト。要素が「ふっくら」している。
5. **ピル型の小バッジ**（「起業」「学ぶ」「触れる」/ 小さな "Recommend" のゴールド小文字）。
6. **アーチ型イラストカード**（カード上辺がアーチ＝門のような形）で入口を見せる。
7. **濃色CTAパネル**＝深い緑〜チャコールの角丸ボックスに白文字、"無料" タグ、**ゴールドの丸ピルボタン＋矢印**。
8. **POINT / FEATURE カード**＝小さな丸 "POINT" バッジ＋大きな斜体数字 "FEATURE 01"、太字タイトル、本文。
9. **「こんな方へおすすめ」ボックス**＝淡い枠＋見出しバー＋緑チェックの箇条書き。
10. **見出しの英語ミニラベル**（"Social Entrepreneur's Program" 等のゴールド小文字キャップ）。
11. **フラットで親しみのあるイラスト**を、空箱プレースホルダの代わりに置く。

## 3. 新しいデザイントークン（CSS変数）

> これを `src/app/globals.css` の `:root` に反映する。**既存の brand-* は活かしつつ、体温を上げる方向に調整**。
> 正式ブランドガイド入手後に上書きする前提（暫定値）。**変えたくなったらこの節の値だけ触れば全体に効く**。

### 3.1 カラー（参考寄りにやわらかく／アンバー・グリーンはブランド軸として残す）

```css
:root {
  /* --- 下地（クリーム系に体温UP） --- */
  --brand-base:   #faf6ee;   /* オフホワイト→ややクリーム寄りに */
  --brand-cream:  #f3ebdd;   /* セクション交互敷き用の一段濃いクリーム */
  --brand-paper:  #ffffff;   /* カード等の純白 */

  /* --- 文字・主要色（黒に振りすぎず、温かいチャコール） --- */
  --brand-ink:    #2a2722;   /* 文字主色：ニュートラルすぎない温かい墨 */
  --brand-ink-2:  #6b6358;   /* 補助テキスト（ウォームグレー） */

  /* --- アクセント（ブランド軸） --- */
  --brand-amber:    #e0852f; /* 灯籠アンバー（既存維持） */
  --brand-amber-d:  #c06a18; /* 同・濃 */
  --brand-gold:     #c8973f; /* ★新規：バッジ／英語ラベル用の落ち着いたゴールド */
  --brand-green:    #3f6b5a; /* 自然のグリーン（既存維持） */
  --brand-green-d:  #2c4f42; /* ★新規：濃色CTAパネル用のディープグリーン */

  /* --- やわらか化トークン --- */
  --radius:       1rem;      /* 角丸を 0.75rem → 1rem に拡大（ふっくら） */
  --shadow-soft:  0 10px 30px -12px rgba(42,39,34,0.18);   /* カード標準影 */
  --shadow-lift:  0 18px 42px -14px rgba(42,39,34,0.26);   /* ホバー浮上 */
  --ring-tint:    rgba(200,151,63,0.35);                   /* フォーカス/縁のゴールド薄膜 */
}
```

### 3.2 セクション背景のリズム

機能羅列に見せないため、**背景を3トーンで回す**：
`cream（下地）→ paper（白カード面）→ tint（淡いセージ or 淡いアンバー 5%）` をセクションごとに交互。
→ `Section` コンポーネントに `tone="cream" | "paper" | "tint-green" | "tint-amber"` を持たせる方針。

### 3.3 タイポ

- 見出し：太め（font-black）＋行間 1.4＋字間ややゆったり。日本語は Noto Sans JP。
- 英語ミニラベル：Inter、小文字 or 小キャップ、`tracking` 広め、色は `--brand-gold`。
- 数字（FEATURE 01 等）：Inter の斜体・極太を装飾的に大きく使う。
- 本文：16px 以上、行間 1.8 前後。

## 4. コンポーネント語彙（参考DNA → 実装名）

Phase1 で「部品」として用意・再利用するもの。**新規 UI は原則ここの語彙に寄せる**。

| 名前 | 役割 | 質感の要点 |
|---|---|---|
| `Eyebrow`（改修） | 英語ミニラベル | ゴールド小文字、`tracking` 広め |
| `Pill` | 小バッジ（「主役」「起業」「無料」等） | 角丸フル、淡い地＋濃い文字、小さめ |
| `SoftCard`（改修 Card） | 標準カード | 角丸 1rem、`--shadow-soft`、ホバーで `--shadow-lift`＋微小 translateY |
| `MediaFrame` | 写真枠（空箱の置換） | `aspect-*`、角丸、影、`object-cover`、フリー素材URL＋差し替えコメント |
| `StatMedallion` | 実績メダル | 円 or 月桂樹風の囲み、大きな数字＋小ラベル、ゴールド |
| `FeatureCard` | POINT/FEATURE | 丸 "POINT" バッジ＋斜体大数字＋太字＋本文 |
| `FitBox` | こんな方へおすすめ | 淡枠＋見出しバー＋緑チェック箇条書き |
| `CtaPanel` | 濃色CTAパネル | ディープグリーン地、白文字、"無料"ピル、ゴールド丸ボタン＋矢印 |
| `ArchCard`（任意） | アーチ型入口カード | カード上辺をアーチ（`border-radius` 上部大）＋イラスト |

## 5. 写真・イラストの仮当て（フリー素材）

- 本番写真が来るまで **フリー素材で必ず「枠」を埋める**（空箱・破線は禁止）。
- 仮素材は `picsum.photos`（例：`https://picsum.photos/seed/yamaga-hero/1200/700`）や
  Unsplash Source 系を使い、**`{/* TODO: 差し替え — 施設写真（廃校外観/灯籠内装/体育館 等） */}` コメントを必ず添える**。
- すべて `alt` 必須、`loading="lazy"`（ヒーローのみ eager 可）。
- 差し替え対象が一覧で分かるよう、画像には `data-placeholder="true"` を付ける。

---

## 6. ぜったい守る（プロジェクト既存ルールの再掲）

- **構成・順番・確定コピーは Phase1 では変えない**（質感のみ）。変えるときは `docs/decision-log.md` に追記。
- 未確認の数字・料金・社名・実績は**創作しない**。`{{要確認:...}}` プレースホルダのまま見た目だけ整える。
- 既存ブランド資産（bY、be you / beyond you / beside you、六角形、「秘密基地」、灯籠）を尊重。
- CTA は問い合わせ／資料請求／視察予約に収束。複数CTAを散らさない。
- アクセシビリティ：コントラスト 4.5:1 以上、見出し階層、alt、フォーカス可視。
- パフォーマンス：重いライブラリ禁止。画像最適化。`prefers-reduced-motion` 尊重。

---

## 7. やりすぎ防止（Don't）

- イラスト・装飾・影を**盛りすぎない**。経営者向けの品を最優先（子どもっぽさNG）。
- 参考サイトの**配色をそのまま使わない**（YAMAGA BASE のアンバー／グリーンが主、ゴールド／クリームは脇）。
- アニメーションは**フェードアップ程度**に留める。動きで内容を邪魔しない。
- 1コミット＝1テーマ。トークン変更とセクション改修を混ぜない（後で戻しやすく）。
